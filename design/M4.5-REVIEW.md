### Worth addressing during M5–M7 (cheap fixes, propagate now)

1. **Contract §1 input table omits `Resume`.** `01-sub-agent-contract.md` §1 lists Issue/Depth/Retry/Branch/PR per stage, but the per-stage rows in §1 do NOT include `Resume` even though §3/§6 rely on it and §2's stage_analyze row in the contract treats `OK PAUSE` as Resume-triggered. M4 had to add `Resume` itself (stage_analyze.md §2, lines 17, 54–60). Add `Resume: <none|continue-after-escalation>` to the §1 input table as a required field across all 4 stages. Also: `Retry` listed in §1 is now misleading — retry is stage-internal per §5, not a main-session input. Suggest dropping `Retry` from §1 entirely.

2. **6th return surfaced for analyze: `OK PAUSE`.** Contract §2's stage_analyze table lists `OK PAUSE` but stage_analyze.md never returns it. analyze.md (line 86) handles `OK PAUSE` on the first-spawn return, but it can only come from the post-escalation re-spawn when the user chose Pause then later resumes. The actual flow in analyze.md is: ESCALATE → AskUserQuestion → Pause → main prints "Resume later" and exits. There is no `OK PAUSE` return path. The contract table is slightly miscoded — `OK PAUSE` is effectively dead in stage_analyze. Either remove it from §2 or add a comment explaining it only appears after a user explicitly invokes `/sdd resume` post-Pause and the resumed sub-agent immediately exits because the user did not pick Continue. Recommend removing for clarity.

3. **`OK ADVANCE` is emitted twice on `continue-after-escalation`.** stage_analyze.md §2 short-circuit re-validates markers exist then returns `OK ADVANCE: design`. analyze.md re-spawns on user Continue, gets `OK ADVANCE: design`, and runs the SAME label transition + skip-review branch again (lines 102–103). On a clean Issue this is idempotent (label is already `sdd:analyze` → moved to `sdd:design`; second pass is moot since label already moved). But if user changes label between escalate and resume, the `--remove-label "sdd:analyze"` is a no-op and `--add-label "sdd:design"` clobbers whatever state the user manipulated. The behavior is reasonable (user's manual change is overwritten by the resume) but undocumented. Add a one-line note to analyze.md ESCALATE → Continue branch.

### Reviewer independence (Q6) — verdict: sufficient, with one caveat

Re-fetching the analyze output per reviewer (lines 253–259, 325, 343) is the right primitive. The risk is not context leakage — it's that the sub-agent's narrative naturally accumulates "Reviewer 1 said X" inside its own context window. Mitigation in stage_analyze.md line 245 + 531 is explicit prose discipline. Strong enough for v1.0.0. If a future audit shows reviewers parroting each other, add a "reset" instruction between reviewers ("After §4.1 completes, treat §4.2 as a fresh evaluation; do not reference §4.1's verdict in your reasoning.").

### Bootstrap interaction (Q4) — duplicated work is acceptable

analyze.md does a label check (lines 17–28) even though M3's bootstrap.md does the same when invoked via `/sdd resume|auto|batch`. The duplicate is intentional per `01-sub-agent-contract.md` §11 — `/sdd analyze <N>` direct invocation skips bootstrap, so the check has to live in analyze.md. The 1-line `gh issue view --json labels` is cheap (~250 tokens) and prevents wrong-stage execution. Apply the same pattern to design.md, implement.md, test.md (each refuses if label doesn't match its stage). Recommend a one-paragraph helper in `_bash_rules.md` or a new `_direct_invocation_check.md` to DRY this across all 4 command wrappers — but if the duplication is just 12 lines × 4 files, inline is fine.

### Skip-review handling (Q9) — the split is correct

stage_analyze handles `skip-review.analyze` (escalation auto-continue, §7 Step 3). analyze.md handles `skip-review.design` (post-OK auto-advance to design, lines 67–72). This mirrors the boundary in `01-sub-agent-contract.md` §4: the sub-agent owns its OWN gate (the escalation prompt); main owns the NEXT-stage gate. Apply identically:
- stage_design owns `skip-review.design`; design.md owns `skip-review.implement`.
- stage_implement owns BOTH `skip-review.implement` (step exhaustion + parent stop) and `skip-review.pr` (PR Final escalation); implement.md owns `skip-review.qa` (auto-advance to test).
- stage_test owns `skip-review.qa`; test.md owns nothing downstream (post-done, no next stage).

### Direct vs auto/batch invocation (Q10) — consistent

Both paths converge on the same Agent spawn shape (`subagent_type: general-purpose`, `model: opus`, same prompt template). Differences are only at the wrapper layer (label check + Phase 0 depth derivation in direct path; bootstrap output supplies these in resume path). Stage sub-agent is invocation-agnostic. Good.
